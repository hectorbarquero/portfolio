My XML pre-processor

I wrote this tool to help a technical writer who was trying to change .xml id tagsets to a placeholder value, but wanted to leave all other document tags unchanged.

My program is lightweight. Try it out.

 

1. Start with a source.xml file

Use your own .xml file, or see how it works with my demo file. If you use your own, just rename it to source.xml for the program to work. My program will reject any file type without an .xml extension, and throw an error.

   

2. Use node.js

I said it was light weight, and it is. 🏃

My program runs from terminal, but you need node.js to run it. Check your version to see if you have it, or download it here.

   

3. Run the program to target id tags

node preProcXML.js

That's it. That's all it takes to run my program. You'll see a success prompt and you can take a look at the output. By default, every valid xml tag is ignored and my regex switches the id tag to a placeholder value of @@

You can change this to suit your needs, but most technical writers will be able to take this output and manipulate it further, doing things like localizing, sanitizing, or batch changes.

   

4. Restore the .xml

After you've manipulated the output, you may want to put humpty dumpty back together again.

node postprocXML.js

That generates a file called finalXML.xml, which really is the same as the source.xml we start with. Don't trust it? I don't blame you. For sanity, I coded a feature that checks.
